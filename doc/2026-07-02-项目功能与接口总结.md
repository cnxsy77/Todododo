## 一、项目概览

| 项目 | 说明 |
|---|---|
| 技术栈 | React Native 0.85 + Expo 56 + TypeScript 6 |
| 导航 | expo-router（Tabs + Stack） |
| 状态管理 | Zustand |
| 数据库 | expo-sqlite（新版 Promise API `openDatabaseAsync`） |
| 动画/手势 | react-native-reanimated、react-native-gesture-handler、react-native-draggable-flatlist |
| 日期 | date-fns |
| 持久化 | AsyncStorage（主题、FAB 位置）+ SQLite（业务数据） |
| 平台 | ✅ Android / iOS 完整可用；⚠️ Web 仅 UI 预览（expo-sqlite 降级返回空结果） |

三大模块：**任务管理**、**货币记账**、**设置**。

## 二、已实现功能与接口

### 2.1 任务管理

#### 功能点
- **多粒度计划**：日 / 周 / 月 / 年四种 `planType`，时间轴 `TimeAxis` 可切换视图。
- **多选分页**：周视图多选周、月视图多选月、年视图多选年（最多 3 个范围），多选提示 UI 已修复多余字符 bug。
- **拖拽排序**：`DraggableFlatList` 实现同日期内自由拖动排序。
- **跨日期拖拽**：支持把任务拖拽到其他日期/范围，已修复残影、错位、顺序丢失、无效落点、回弹等问题（提交 `439bed2`，引入 `localRenderData` 本地立即更新，参考 memory `draggableflatlist-single-data-change`）。
- **可拖动添加按钮**：`MovableFab`，拖动后吸附左右边缘，位置持久化到 AsyncStorage，轻点触发新增（提交 `1638162`）。
- **任务详情**：标题、描述、计划类型、开始日期、完成切换、删除。

#### 接口（`src/database/queries.ts`）
| 函数 | 说明 |
|---|---|
| `getAllTasks()` | 查询全部任务 |
| `getTaskById(id)` | 按 id 查询（已实现，当前 UI 未直接调用） |
| `createTask(input)` | 新建任务，自动取 `MAX(sort_order)+1` |
| `updateTask(id, input)` | 更新任务字段（标题/描述/planType/日期/完成/排序/父任务） |
| `deleteTask(id)` | 删除任务 |
| `updateTaskOrders(planType, taskIds)` | 按传入顺序批量重排 `sort_order` |
| `moveTaskToDate(id, newStart, newEnd?)` | 移动任务到新日期（仅改 `start_date/end_date`） |

#### Store（`src/stores/taskStore.ts`）
`loadTasks` / `addTask` / `updateTask` / `deleteTask` / `toggleTaskCompleted` / `reorderTasks` / `moveTaskToDate` / `moveTaskToDateWithOrder`（合并移动+重排，避免两次 set 造成 DraggableFlatList 抖动）。

#### Hooks（`src/hooks/`）
- `useView()` / `useViewSwitcher()`：视图、当前日期、选中范围、上/下一切换。
- `useTasksByRanges(ranges)`：按选中时间范围分组加载任务。

### 2.2 货币记账

#### 功能点
- **交易记录**：收入/支出，金额、分类、备注、日期。
- **交易编辑与删除**：详情页支持编辑（动态标题"添加/编辑记录"）与确认删除（06-22 标记的缺失已修复）。
- **分类管理**：内置 14 个默认分类（5 收入 + 9 支出），支持在交易详情页内**新建分类**（名称/图标/颜色，Modal 选择）。
- **统计**：`StatisticsScreen`，周/月/年切换，收支总额、分类占比、趋势。
- **统计入口**：货币页快捷按钮跳转 `/statistics`（06-22 标记的"无入口"已修复）。
- **预算管理**：`BudgetScreen` 完整 UI——本月总览进度条、按支出分类设置月/年预算、已用/总额、超支高亮、编辑/删除。

#### 接口
- `src/database/transactionQueries.ts`：交易 CRUD + `getAllTransactions` + 统计聚合。
- `src/database/categoryQueries.ts`：`createCategory` / 分类查询（**缺 `updateCategory`**）。
- `src/database/budgetQueries.ts`：预算完整 CRUD。
- Stores：`transactionStore`（含 `getStatistics`）、`categoryStore`（`loadCategoriesByType` / `addCategory`）、`budgetStore`（`loadBudgets` / `addBudget` / `updateBudget` / `deleteBudget`）。

### 2.3 设置与数据

#### 功能点
- **主题**：亮色 / 暗色 / 跟随系统三态；`loadTheme()` 启动加载（已在 `app/_layout.tsx` 调用）；`auto` 模式注册 `Appearance.addChangeListener` 跟随系统变化；色板 `lightColors/darkColors` 通过 `useTheme()` 全局应用至所有页面与组件（06-22 标记的"主题未应用"已全面修复）。
- **数据导出**：`exportData()` 汇总 tasks/transactions/categories/budgets 为 JSON，写入文档目录并通过 `expo-sharing` 分享。
- **数据导入**：`importData(fileUri)` 通过 `expo-document-picker` 选文件 → 校验 → 事务内清空四表 → 批量插入 → 返回 `ImportSummary` → 刷新全部 store（06-22 标记的"未实现"已完整修复）。
- **关于**：GitHub 链接、版本号。

---

### 2.4 基础设施

- **数据库 Schema**（`src/database/schema.ts`）：已迁移到新版 `openDatabaseAsync` Promise API；`useNewConnection: true` 规避 dev tools 注册干扰 SharedObject 生命周期导致的 Android NPE（提交 `1b29895`，参考 memory `expo-sqlite-android-npe`）。单例 `getDatabase()`。
- **迁移系统**（`src/database/migrations.ts`）：版本化，`CURRENT_VERSION = 2`。v1 建 `tasks` 表 + 索引；v2 建 `transactions` / `categories` / `budgets` 表 + 索引 + 默认分类。Web 端跳过。
- **路由**（`app/`）：Tabs（任务 `index` / 货币 `currency` / 设置 `settings`）+ Stack（`task-detail` / `transaction-detail` / `statistics` / `budget`）。
- **主题系统**：`ThemeColors` 接口 + light/dark 色板 + `useTheme()` hook。
- **Web 降级**：`Platform.OS === 'web'` 各处跳过数据库操作，README 已声明 Web 仅 UI 预览。

## 三、未实现的接口和功能

### 3.1 数据模型已存在但 UI 未接入

| 项 | 现状 |
|---|---|
| **子任务/层级** | `tasks.parent_task_id` 字段、`Task.parentTaskId` 类型、`CreateTaskInput.parentTaskId` 均存在，但任务详情页无父任务选择/子任务创建入口，`queries.ts` 无 `getChildTasks`，层级功能完全不可用。 |
| **任务结束日期 `endDate`** | 类型与 schema 支持，详情页只编辑 `startDate`，无 `endDate` UI（周/月/年计划实际无法设定跨日期范围）。 |
| **任务搜索/筛选** | 无搜索接口与 UI；列表页无法按关键字、完成状态、分类筛选。 |
| **分类修改** | 缺 `updateCategory`，分类只能删除重建。 |
| **交易列表筛选** | 列表页仅按日期分组展示，无法按分类/月份/收支类型筛选。 |

### 3.2 已知缺陷 / 逻辑漏洞

| 项 | 现状 | 影响 |
|---|---|---|
| **`moveTaskToDate` 不更新 `plan_type`** | 仅改 `start_date/end_date` | 将 daily 任务拖到 weekly 范围后，切到周视图按 `planType` 过滤时该任务可能消失（06-22 已记录，至今未修）。 |
| **`updateTaskOrders` 全量重排** | 分组模式同日期重排分支把所有范围的任务 ID 传给 `onReorder`，对全部重新编号 | 重载后可能打乱其他日期的排序（06-22 已记录）。 |
| **统计页无图表** | `StatisticsScreen` 仅文本/列表展示，未引入图表库 | 占比/趋势可视化缺失。 |
| **Web 端数据库不可用** | expo-sqlite Web 降级返回空 | Web 端业务功能不可用（已声明，预期降级）。 |

### 3.3 完全缺失的功能

- **任务提醒/通知**：无 `expo-notifications` 接入，无到期提醒。
- **重复任务**：无重复规则（每日/每周/每月）。
- **数据云同步 / 多设备**：仅本地 SQLite，无账号、无云备份。
- **定时自动备份**：导出需手动触发。
- **任务附件/富文本描述**：描述为纯文本 `TEXT` 字段。
- **国际化 i18n**：所有文案硬编码中文。
- **测试体系**：无单元测试、无 E2E，`package.json` 无 `test` 脚本。
- **大量任务分页/虚拟化优化**：`getAllTasks` 一次性全量加载。
- **iOS 真机验证**：近期修复集中在 Android/Web，iOS 未验证。

## 四、下一步推荐方向

按"修复缺陷 → 补齐核心 → 体验增强 → 工程化"排序：

### P0 修复缺陷（低成本、防数据错乱）
1. **`moveTaskToDate` 同步 `plan_type`**：跨视图拖拽时按目标范围更新 `plan_type`，避免任务"消失"。同时审视 `updateTaskOrders` 全量重排，改为仅重排受影响日期。
2. **任务 `endDate` UI**：周/月/年计划允许选择结束日期，让跨日期范围任务可建可改。

### P1 补齐核心功能
3. **子任务层级**：详情页支持挂载父任务、展开子任务列表，`queries` 补 `getChildTasks` / `getTaskWithChildren`，列表组件支持折叠展示。
4. **分类修改 `updateCategory`**：补 store + queries + 编辑入口。
5. **任务搜索与筛选**：列表页加搜索框与状态/分类/日期筛选。

### P2 体验增强
6. **任务提醒**：接入 `expo-notifications`，任务可设提醒时间，本地推送。
7. **重复任务**：新增重复规则字段，到期自动生成下一周期任务。
8. **统计可视化**：引入图表库（如 `react-native-chart-kit` / `victory-native`），分类占比饼图、趋势折线图。
9. **定时自动备份**：每日首次启动自动导出 JSON 到本地。

### P3 工程化与扩展
10. **测试体系**：为核心 queries/stores 加单元测试（jest + 内存 sqlite），关键流程 E2E。
11. **国际化 i18n**：抽离文案，为多语言做准备。
12. **数据云同步**：账号体系 + 云端备份/同步（Firebase / 自建后端）。
13. **iOS 真机验证**：跑通 iOS 构建，确认 expo-sqlite 与手势在 iOS 表现一致。
14. **性能优化**：任务分页加载、`FlashList` 替换、大量数据下虚拟化。
