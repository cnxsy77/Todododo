## 二、已实现功能与接口

### 2.1 任务管理

#### 功能点
- **多粒度计划**：日 / 周 / 月 / 年四种 `planType`，时间轴 `TimeAxis` 可切换视图。
- **多选分页**：周视图多选周、月视图多选月、年视图多选年（最多 3 个范围），多选提示 UI 已修复多余字符 bug。
- **拖拽排序**：`DraggableFlatList` 实现同日期内自由拖动排序。
- **跨日期拖拽**：支持把任务拖拽到其他日期/范围，已修复残影、错位、顺序丢失、无效落点、回弹等问题（提交 `439bed2`，引入 `localRenderData` 本地立即更新，参考 memory `draggableflatlist-single-data-change`）。
- **可拖动添加按钮**：`MovableFab`，拖动后吸附左右边缘，位置持久化到 AsyncStorage，轻点触发新增（提交 `1638162`）。
- **任务详情**：标题、描述、计划类型、开始日期、结束日期（可选，可清除）、完成切换、删除。
- **任务结束日期 `endDate`**：详情页可设定/清除结束日期（`DatePicker` 增强支持 undefined + 清除），卡片在 endDate 与 startDate 非同日时显示「起 - 止」范围。简化版：列表仍按 startDate 归属单一 range，跨范围任务不在覆盖的其他 range 重复显示（跨范围显示需重构 TaskList，参考 memory `task-enddate-no-cross-range`）。
- **任务搜索与筛选**：首页搜索框按标题/描述关键字过滤 + 完成状态 chips（全部/未完成/已完成）；仅作用于父任务，子任务跟随父显示。筛选态禁用拖拽（避免部分可见任务重排破坏全局顺序）。
- **任务提醒/通知**：`expo-notifications` 本地推送。任务详情可设定提醒时间 `reminderAt`，`createTask`/`updateTask` 经 `notificationService.scheduleTaskReminder` 调度（identifier 复用 taskId 便于取消，已过期时间不调度）；完成时取消该任务及其子任务提醒，删除时级联取消（提交 `4a4541d`）。
- **按视图精确加载**：首页列表不再 `getAllTasks` 全量加载，改用 `getTasksByPlanTypeAndRanges(planType, ranges)` 仅查当前视图范围内任务；store 仅持有当前 range+planType 任务，写操作据此加入/移除（提交 `1add184`，参考 memory `task-list-view-scoped-loading`）。
- **Markdown 富文本描述**：任务描述支持 Markdown，详情页用 `marked` 解析 + 自写 `MarkdownRenderer` 渲染（标题/粗斜体/代码/列表/引用/链接/图片等），卡片单行预览用 `stripMarkdown` 去语法（提交 `96980b5`，参考 memory `markdown-rendering-marked`）。底层 `description` 仍为 `TEXT` 存原始 md。

#### 接口（`src/database/queries.ts`）
| 函数 | 说明 |
|---|---|
| `getAllTasks()` | 查询全部任务（仅供导出等全量场景） |
| `getTasksByPlanTypeAndRanges(planType, ranges)` | 按计划类型 + 日期范围精确加载（首页视图按需查询，替代全量）；空 range 返回 [] |
| `getTaskById(id)` | 按 id 查询（已实现，当前 UI 未直接调用） |
| `createTask(input)` | 新建任务，自动取 `MAX(sort_order)+1`；指定父任务时强制继承父的 planType/startDate/endDate 并校验仅两层 |
| `updateTask(id, input)` | 更新任务字段（标题/描述/planType/日期/完成/排序/父任务）；`endDate` 传 `null` 表示清除（写 SQL NULL），`undefined` 表示不更新；父任务日期/planType 变更级联子任务 |
| `deleteTask(id)` | 删除任务（级联删除其所有子任务） |
| `setTaskAndChildrenCompleted(id, completed)` | 设置任务及其所有子任务的完成状态（完成联动） |
| `updateTaskOrders(planType, taskIds)` | 按传入顺序批量重排 `sort_order` |
| `moveTaskToDate(id, newStart, newEnd?)` | 移动任务到新日期（改 `start_date/end_date`，级联子任务保持同 range） |

#### Store（`src/stores/taskStore.ts`）
`loadTasks` / `addTask` / `updateTask` / `deleteTask` / `toggleTaskCompleted` / `reorderTasks` / `moveTaskToDate` / `moveTaskToDateWithOrder`（合并移动+重排，避免两次 set 造成 DraggableFlatList 抖动）。

#### Hooks（`src/hooks/`）
- `useView()` / `useViewSwitcher()`：视图、当前日期、选中范围、上/下一切换。
- `useTasksByRanges(ranges, planType)`：按选中时间范围 + 当前视图 planType 加载任务，`useEffect` 按需查库，`useMemo` 同步派生。内部调用 `buildTree`（已导出为纯函数，供测试直接调用）把扁平列表组装成父子树并按 range 分组。

### 2.2 货币记账

#### 功能点
- **交易记录**：收入/支出，金额、分类、备注、日期。
- **交易编辑与删除**：详情页支持编辑（动态标题"添加/编辑记录"）与确认删除（06-22 标记的缺失已修复）。
- **分类管理**：内置 14 个默认分类（5 收入 + 9 支出），支持在交易详情页内**新建分类**（名称/图标/颜色，Modal 选择）；**长按分类卡片可编辑**分类名称/图标/颜色（`updateCategory` UI 接入，07-14 补齐）。
- **统计**：`StatisticsScreen`，周/月/年切换，收支总额、分类占比、趋势。
- **统计入口**：货币页快捷按钮跳转 `/statistics`（06-22 标记的"无入口"已修复）。
- **预算管理**：`BudgetScreen` 完整 UI——本月总览进度条、按支出分类设置月/年预算、已用/总额、超支高亮、编辑/删除。
- **交易列表筛选**：货币页顶部筛选条，支持收支类型（全部/收入/支出）、月份切换（左右翻月 + 全部月份）、分类（横向 chip 行，随类型联动）；余额卡片保持全量统计，筛选只作用于列表（07-14 补齐）。

#### 接口
- `src/database/transactionQueries.ts`：交易 CRUD + `getAllTransactions` + 统计聚合。
- `src/database/categoryQueries.ts`：`createCategory` / `updateCategory`（07-14 补齐）/ 分类查询。
- `src/database/budgetQueries.ts`：预算完整 CRUD。
- Stores：`transactionStore`（含 `getStatistics`）、`categoryStore`（`loadCategoriesByType` / `addCategory`）、`budgetStore`（`loadBudgets` / `addBudget` / `updateBudget` / `deleteBudget`）。

### 2.4 基础设施

- **数据库 Schema**（`src/database/schema.ts`）：已迁移到新版 `openDatabaseAsync` Promise API；`useNewConnection: true` 规避 dev tools 注册干扰 SharedObject 生命周期导致的 Android NPE（提交 `1b29895`，参考 memory `expo-sqlite-android-npe`）。单例 `getDatabase()`。
- **迁移系统**（`src/database/migrations.ts`）：版本化，`CURRENT_VERSION = 2`。v1 建 `tasks` 表 + 索引；v2 建 `transactions` / `categories` / `budgets` 表 + 索引 + 默认分类。Web 端跳过。
- **路由**（`app/`）：Tabs（任务 `index` / 货币 `currency` / 设置 `settings`）+ Stack（`task-detail` / `transaction-detail` / `statistics` / `budget`）。
- **主题系统**：`ThemeColors` 接口 + light/dark 色板 + `useTheme()` hook。
- **Web 降级**：`Platform.OS === 'web'` 各处跳过数据库操作，README 已声明 Web 仅 UI 预览。
- **Markdown 富文本描述**：`marked` 解析 + 自写 `MarkdownRenderer`（`src/components/MarkdownRenderer`）渲染任务描述，`MarkdownToolbar` 提供插入语法快捷键；`src/utils/markdown.ts` 的 `stripMarkdown` 用于卡片单行预览去语法。未引入停更的 `react-native-markdown-display`（参考 memory `markdown-rendering-marked`）。
- **测试体系**：Jest + jest-expo 单元测试（不含组件渲染与 E2E）。
  - 配置：`jest.config.js`（preset `jest-expo`，testMatch `**/test/**/*.test.ts(x)`，`transformIgnorePatterns` 放行 `marked`/`date-fns` 等 ESM 包）；`test/jest.setup.ts` 全局 mock AsyncStorage / expo-notifications；`test/jest.d.ts` 引入 jest 全局类型。
  - 结构：`test/` 下 `utils/`（markdown / uuid / date / export-import）、`stores/`（task / transaction / category / budget / theme / view）、`database/`（queries / transactionQueries / categoryQueries / budgetQueries）、`hooks/`（useTasks.buildTree）共 15 个测试文件、179 用例。
  - Mock 策略：stores 测试 mock 各 `database/*Queries`，zustand 真实跑测状态机；database 测试 mock `getDatabase` 返回 fake db（jest.fn），验证 SQL 构建字符串 / 参数绑定顺序 / `mapRow` 列映射 / 级联逻辑，不验证 SQL 执行语义；utils export/import 测试 mock `expo-file-system` / `expo-sharing` / queries 编排与 `ImportSummary`。
  - 命令：`npm test` / `npm run test:watch` / `npm run test:coverage`。覆盖率：utils / stores / database 核心逻辑高覆盖（markdown / uuid / export / viewStore 100%，整体约 87% 语句覆盖）。
  - 为可测性：`src/hooks/useTasks.ts` 导出 `buildTree` 纯函数（无行为变化）。
