# 跨日期拖拽问题修复报告

## 问题描述

在 Todododo 应用中，多选日期场景下跨日期拖拽任务存在以下问题：

1. **拖拽后任务回到原位置**：将任务从一个日期拖到另一个日期后，松手时任务会跳回原来的位置
2. **拖到任务下方无效**：将任务 A 拖到任务 B 下面时，松手后任务 A 出现在任务 B 前面而不是后面

## 问题根源分析

### 问题 1：数据同步时机问题

拖拽流程：
1. `handleDragEndWithHeaders` 被调用
2. `setLocalRenderData` 和 `setIsUsingLocalData(true)` 更新本地状态
3. `onMoveTaskToDate` 异步更新 store
4. store 更新完成后，`tasks` prop 变化
5. `useEffect` 被触发，**无条件重置**本地数据为 props 数据
6. UI 显示被覆盖，任务"跳回"原位置

**关键问题**：`useEffect` 在 `setLocalRenderData` 后立即触发，覆盖了正确的本地数据。

### 问题 2：插入位置计算错误

原有逻辑根据 `to-1` 位置的项目判断插入点，但索引映射复杂且易错：
- `newData` 是拖拽后的数组（包含被拖拽任务在 `to` 位置）
- `newLocalData` 是移除 `to` 位置后的数组
- 两个数组的索引需要正确映射

**关键问题**：没有正确区分"拖到任务上面"和"拖到任务下面"两种场景。

## 修复方案

### 修复 1：使用 `pendingSyncRef` 防止数据覆盖

```typescript
const pendingSyncRef = React.useRef(false);

// 拖拽后设置标记
setIsUsingLocalData(true);
pendingSyncRef.current = true;

// useEffect 检查标记
React.useEffect(() => {
  if (pendingSyncRef.current) {
    return; // 跳过，不覆盖本地数据
  }
  // ... 重置数据
}, [tasks, renderDataFromProps]);

// 延迟重置标记
setTimeout(() => {
  pendingSyncRef.current = false;
}, 100);
```

### 修复 2：正确的插入位置计算

```typescript
// 在 newLocalData 中找到最后一个目标范围任务的本地索引
let lastTargetTaskLocalIndex = -1;
for (let i = newLocalData.length - 1; i > targetHeaderIndex; i--) {
  const item = newLocalData[i];
  if (!('isHeader' in item) &&
      (item as DraggableTask).rangeStart === targetRange.start) {
    lastTargetTaskLocalIndex = i;
    break;
  }
}

// 在 newData 中找到最后一个目标范围任务的索引（排除被拖拽任务）
let lastTargetTaskNewDataIndex = -1;
for (let i = newData.length - 1; i >= 0; i--) {
  const item = newData[i];
  if (!('isHeader' in item) &&
      (item as DraggableTask).rangeStart === targetRange.start &&
      i !== to) {
    lastTargetTaskNewDataIndex = i;
    break;
  }
}

// 根据 to 和 lastTargetTaskNewDataIndex 的关系确定插入位置
if (to > lastTargetTaskNewDataIndex) {
  // 拖到了最后一个任务后面
  insertIndex = lastTargetTaskLocalIndex + 1;
} else {
  // 拖到了最后一个任务前面
  insertIndex = firstTargetTaskLocalIndex;
}
```

## 修复场景验证

| 场景 | from | to | lastTargetTaskNewDataIndex | 条件 | 插入位置 | 结果 |
|------|------|-----|---------------------------|------|---------|------|
| 拖到 T3 上面 | 1 | 3 | 3 | to < last | T3 前面 | ✓ |
| 拖到 T3 下面 | 1 | 4 | 3 | to > last | T3 后面 | ✓ |

## 修改文件

- `src/components/TaskList/index.tsx`

## 验证步骤

1. 选择多个日期（例如 24 日、25 日）
2. 确保 24 日有任务 1 和任务 2，25 日有任务 3
3. 拖拽任务 1 到 25 日任务 3 **上面** → 任务 1 应在任务 3 前面
4. 拖拽任务 1 到 25 日任务 3 **下面** → 任务 1 应在任务 3 后面
5. 拖拽任务 1 回 24 日任务 2 **下面** → 任务 1 应在任务 2 后面，且仍在 24 日

## 技术要点

1. **使用 `useRef` 而非 `useState`**：`pendingSyncRef` 不需要触发重新渲染，只需在闭包中保持最新值
2. **延迟重置标记**：使用 `setTimeout` 延迟 100ms，确保 UI 有足够时间使用本地数据渲染
3. **双重索引查找**：分别在 `newLocalData` 和 `newData` 中查找目标范围任务，确保索引映射正确
